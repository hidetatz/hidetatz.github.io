Claude Codeという便利なソフトウェアがある。私も使っている。使っているんだけど、プログラミングはほとんどさせてない。自分で書くのが単に面倒な、小さなコードのピース (例えば、Pythonのstructモジュールを使ってバイナリをパックするとか) を作らせるくらいはするけど、それを推敲したり、組み合わせたりするのは自分でやっている。これはあまり一般的な使い方ではないかもしれなくて、なぜこうしてるのか自分でもよくわからないので、考えてみた。考えてみると少し大きな話になってきたので、書きながら思考を整理してみよう。

LLMを中心としたAIっていうのは、そもそも何が便利なんだろうか？質問に答えてくれて、わからないことを教えてくれる。これは間違いなく便利だ。LLMっていうのは文字列を入力するとその文字列の続きを出力するソフトウェアなので、質問への回答ができる。このソフトウェアを無限ループの中に入れて、次のようなコードを書くとAIは自律して行動ができるようになり、それを我々はAIエージェントと呼ぶ。

```Python
while True:
    current_state = collect_current_state()
    next_action = llm_inference(current_state)
    run_action(next_action)
```

AIエージェントにできることは色々あるが、今はプログラミングについて話そう。AIがプログラミングできると何が嬉しいのか？嬉しいことは色々あるけど、一番うれしいのは「速い」ということじゃないだろうか？大抵の人間よりも速くコードを書ける。これに関しては確かにそうで、地球上にAIエージェントより速くプログラミングできる人間は存在しないだろう。人間の時間は有限なので、速いことはほとんど常に良いことだろう。

さて、プログラミングが速いと何が嬉しいのか？というのをもう少し考えよう。ここでは仕事におけるプログラミングを想定する。つまり、ソフトウェアを作って売ったり、ソフトウェアを使う権利を有償・あるいは無償で人々にあげることで収益をあげるような、そんなビジネスでもあるだろうし、あるいはどんな部分でもいいから、なんらかのソフトウェアが業務に組み込まれていて、そのソフトウェアの出来が、多かれ少なかれビジネスの行方を左右するような、そんな環境の話だ。

さて、プログラマーにとって、プログラミングが速く終わると何が嬉しいのか？自分の仕事が速く終わること？でも、特に会社員の人にとっては、別にプログラミングが速く終わると、速く家に帰って寝たりストレッチをしたりできるかというと、そういうわけでもないだろう。プログラミングが速く終わると、単に次のプログラミングがやってくるだけだ。だから、プログラミングが速く終わっても、別に嬉しくないんじゃないか？

もう少し目線を上げて、ソフトウェアを作って、それを人に使わせることを考えよう。使わせるのは顧客でも、隣の席の社員でも別に誰でも良い。
ソフトウェアを作ったことがある人ならわかると思うけど、ソフトウェアを作るときに最も難しいのは、どんなソフトウェアを作るか考えることだ。やりたいことがあっても、そのソフトウェアの見た目や使い方、内部構造などは非常に多くのパターンがあり得るし、現実的には、何がやりたいのかすら曖昧なことも多い。こういうときに取れるのが、「そのときにベストと思われるものをなるべく速く作ってしまい、使わせてから考える」というやり方だ。これはソフトウェア開発のPDCAサイクルの一部で、ほとんど定石と言って良いと思う。

プログラミングが速くできると、このPDCAサイクルが速く回せるから嬉しい、という論理が成り立ちそうだ。でも、これって本当にそうか？例えばソフトウェアを作って売っている会社があるとして、顧客を喜ばせるためにソフトウェアを改善したくて、でもどう改善すればいいかわからないという場合、AIに作らせることで今までよりも速く作れるから嬉しい、と考えるだろう。でも、ここがポイントで、同じような高速PDCAを、できるのは自分たちだけではない。競合他社も同じことができるわけだ。なぜなら、AIエージェント自体は、お金さえ払えば誰でも使えるからだ。

こうなると話は変わってくる。ソフトウェアをとにかく速く作ってPDCAすることで他社に差をつけられると思ったけど、それが起こることはない。アイディア自体をAIに考えてもらうのはどうだろうか？でも同じアイディアを、AIは競合他社にも教えてしまうのだ。

AI自体はお金を払えば誰でも使える、これは動かしがたい事実だ。となると変数となるのは、お金が払えるかどうか？と、上手く使えるかどうか？だろう。競合他社はお金を払えるかどうか？これは心配ない。あなたが払えるのなら、競合他社も払えるだろう。なぜなら、OpenAIやAnthropicは、あなたが死なない程度の、このくらいなら払ってもよいだろうという気持ちになる値付けをよく心得ているからだ。

じゃあ、競合他社はAIを自分たちのように上手く使えるだろうか？私の見解だと、これについて世界中のみんなで一生懸命考えているのが、ちょうど今だ。でも個人的には楽観的に考えていて、つまり、世界中の誰もが、AIをとてもうまく使えるようにすぐになるだろう。なぜならOpenAIやAnthropicはモデルを作るだけではなくて、AIを上手く使うためのソフトウェアを作ることの重要性をとてもよくわかっているからだ。Claude CodeとかClaude Designとか、Codex Sitesとかはそこから生まれたものだ。それに、AIを上手く使う方法は、AI自体がよく知っている。周辺技術も積極的に標準化されて、オープンな形で進歩している。そのうち、AIが作ったソフトウェアをとても簡単にデプロイする機能みたいなのもできるんじゃないだろうか？

つまり何が言いたいかというと、「自分たちのプログラミングだけが速くなる」のであればこれは確かに嬉しいのだけど、現実的には「全ての人間のプログラミングが速くなっている」わけで、もう一度よく考えてみてほしいんだけど、これって何が嬉しいんだろうか？

私がAIエージェントというものに対してここ1年くらいなんとなく抱いていた違和感の正体はおそらくこういうことで、まだ全体像が見えたわけではないんだけど、ようやく言語化することができた。

さて、全ての人間のプログラミングが速くなっているという前提で、じゃあビジネスに置いて差をつけ、Moatになるようなものって一体なんだろうか？これは結局、人間に他ならない。AIは武器にはならない。なぜなら同じ武器を全員持っているからだ。組織の強さはそこに所属する人間の強さだ。これはどんなツールが出てこようが変わらない。100%では決してないが、強い人間が多くいる組織は基本的には勝ちやすい。

じゃあ、AIによって人間は強くなれるだろうか？私はあまりそう思っていなくて、現代のAIエージェントの多くは、多くの人間を怠惰にさせ、依存させ、長期的にはより弱くさせるだろう。我々はAIを、自分が強くなるために使うべきだ。もちろん、自分の仕事さえ終わればそれでいい、みたいな人はそれでもいいんだけど、少しでも何かを実現したいという気持ちがある人は、AIの使い方にはかなり慎重になるべきだと考えている。AIを使って何かを実現した気持ちになっても、他人もそれを簡単にできるとなると、最終的には自分から出てくるもので差をつけるしかない。

私はLLM自体は本当にこの世界や歴史を変える、優れた発明だと思っている。私企業によるサービスとしてのLLMであっても、使わない事自体はほぼありえないくらいに言ってよいだろう。ただ、その適切な使い方を、やはり我々はまだわかっていないのだと考えている。トークンを使わないともったいないとか、会社の中で誰が一番Claudeを使っているからすごいとか、そんなこと言っている場合ではないんじゃないかマジで、という感覚で、この問題をどうすれば解決できるのか考えている。

でも、ちょっとした発明で解決できるとも考えている。LLMというソフトウェアのインタフェースを工夫することで、考え・行動して強くなろうとする人間をより強くし、本当に面倒な作業、例えばPythonのバイナリパッキングとか、パブリッククラウドのIAM設定とか、そういう間違いなく必要だけど面倒な作業、プログラムを書いて自動化するには複雑すぎるけど、LLMならできるような、そういうことだけを的確に代行させるようなソフトウェアが、きっと作れると思っている。私に想像できたのだから、他の人にもきっとできるだろうし、想像できるのなら、実現できるだろう。
